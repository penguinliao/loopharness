# Hermes 自反思提议（待审核）

- **降级路径** — `retreat_log.md` 不存在/为空/损坏时，注入逻辑必须静默跳过而非崩溃，绝不能因读错题本失败把整条 pipeline 卡死（fail-open）。
- **相似度边界** — 复用 antagonist 的 `_reason_similar`/0.85 阈值时，要处理空字符串、None、超短文本（如"P0"两字）这类退化输入，避免除零或误判 100% 相似而提前误升级。
- **注入即物理** — AC1 要求错题本"物理注入"到 IMPLEMENT，必须真写进 brief 文件/prompt 让干净上下文 AI 必读，不能只在 prompt 里写"请先读"——否则又退回违背的「AI 承诺」。
- **提前升级有出口** — 检测到 retreat 原地打转后升级到的状态（stuck/人工介入）必须给 PM 可读的明确原因，而非静默停住让无人值守 /loop 空转。
- **计数与升级不冲突** — 新增"连续卡同一失败即升级"要和已有"数到 3 次"逻辑共存，明确谁先触发，避免两套退出条件互相覆盖。
- **编码与多轮累积** — 错题本多轮 append 后含中文/特殊字符，读回注入时按 UTF-8 处理，且只注入最近 N 条而非全量，防止 context 膨胀。
- **自举可测** — 因 IMPLEMENT 阶段 test_*.py 被物理锁定，相似度/注入的判定函数必须在 SPEC 阶段就能从 spec 推出接口签名（`_retreat_briefing`/`_reason_similar`），测试基于行为而非实现。
